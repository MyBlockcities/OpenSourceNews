## 🚀 Deployment Steps

### Step 1: Connect to Railway

1. Go to https://railway.app
2. Sign in with GitHub
3. Click "New Project"
4. Select "Deploy from GitHub repo"
5. Choose: `MyBlockcities/scheduler`
6. Railway will auto-detect Node.js project

### Step 2: Configure Environment Variables

In Railway dashboard:

1. Click on your service
2. Go to "Variables" tab
3. Add these variables:

```
VITE_GEMINI_API_KEY = your_gemini_api_key_here
```

**Note:** For Vite apps, env variables MUST be prefixed with `VITE_`

### Step 3: Configure Build Settings (Auto-detected)

Railway will automatically use:
```
Build Command: npm run build
Start Command: npm run preview -- --host 0.0.0.0 --port $PORT
```

If not auto-detected, set manually in Settings → Deploy.

### Step 4: Deploy

1. Railway will automatically deploy on push to main
2. Wait for build to complete (~2-3 minutes)
3. Click on generated URL to view your app

**Example URL:** `your-app.up.railway.app`

## ⚠️ Important Notes

### 1. Outputs Directory

The `outputs/` directory contains daily reports generated by GitHub Actions.

**Deployment Option A (Current):**
- Include `outputs/` in git (commit daily reports)
- Railway serves them as static files
- Pro: Simple, works immediately
- Con: Git repo gets larger over time

**Deployment Option B (Recommended Long-term):**
- Store reports in Railway Volume or external storage
- Fetch via API
- Pro: Cleaner git repo
- Con: More complex setup

**For now:** Option A works fine (outputs are small JSON files)

### 2. GitHub Actions Integration

Your current setup:
```
GitHub Actions (automated) → Commits to repo → Railway auto-deploys
```

**This means:**
1. Daily pipeline runs at 7 AM UTC
2. Generates new report JSON
3. Commits to GitHub
4. Railway detects commit and redeploys
5. New reports appear in UI

**Cost:** Free (Railway hobby plan includes auto-deploys)

### 3. Flask API (Optional)

For advanced script generation:

**Option A:** Deploy Flask separately
- Create new Railway service for `api/script_generator.py`
- Update vite.config.ts proxy to point to Railway Flask URL
- Cost: +$5/month for second service

**Option B:** Keep it local (Current)
- Script generation works in browser (free)
- Use Flask locally for advanced features
- Cost: $0

**Recommendation:** Start with Option B (current setup)

## 🐛 Common Issues & Solutions

### Issue 1: Build Fails with "Module not found"

**Cause:** Missing dependencies

**Solution:**
```bash
# Locally verify dependencies
npm install
npm run build

# If successful, commit package-lock.json
git add package-lock.json
git commit -m "Add package-lock.json"
git push
```

### Issue 2: "process is not defined"

**Cause:** Using Node.js process.env in browser

**Solution:** Already fixed in `services/geminiService.ts`
- Changed from `process.env` to `import.meta.env`
- Added `vite-env.d.ts` for TypeScript types

### Issue 3: Daily reports not loading

**Cause:** publicDir not configured

**Solution:** Already fixed in `vite.config.ts`
- Set `publicDir: 'outputs'`
- Reports accessible at `/daily/*.json`

### Issue 4: Port binding error

**Cause:** Not using Railway's PORT variable

**Solution:** Already fixed in nixpacks.toml
- Start command includes: `--port $PORT`
- Railway injects PORT automatically
